JS前端加密绕过

0x00 前言

​ 弱口令是入侵网站最常使用到的一种方式，一般通过burpsuite暴力破解可达到目的。但有些网站使用了前端加密，加大了爆破的难度，很多时候遇到这种情况就放弃了，比较可惜。但是发现越来越多的网站都开始做前端加密了，所以有必要学习一下前端加密的绕过了。

0x01 前期准备

burpsuite

jsEncrypter插件

phantomJS(无头浏览器)

下载好这几样工具后，

1、burpsuite加载插件jsEncrypter。

2、将phantomJS下的\phantomjs-2.1.1-windows\bin加入环境变量。

0x02 模拟测试

借助phpstudy开启web服务，将/test/webapp放到phpstudy里测试demo，然后模拟下真实情况下使用该插件。

以base64的前端加密绕过为例，如图 抓包可以看到密码被加密成了base64格式：

第一步：F12打开调试器查看源代码，找到加密的函数和js。
第二步：将xxx.js下载下来，放到script目录。将实现加密算法的 js 文件引入模板脚本，编写 phantomJS 运行脚本，并在模板脚本的 js_encrypt 函数体中完成对加密函数的调用。phantomjs_server.js修改如下（这里以crypto-js.js为例）：

​ 导入js文件后，编写加密算法再调用其中的依赖。

第三步：切到插件存放的scripts目录，打开命令行运行phantomjs phantomjs_server.js，接下来在burpsuite进行操作。

​ 这其中有个问题就是怎么找到关键的js文件，我们可以通过 ctrl+shift+f 全局搜索 encrypt、aes、des、md5、key、iv、rsa 等关键字，然后导入对应的js。

​ 接下来要找的就是实现对应数据加密的加密函数：

方法一：浏览器的事件监听

方法二：搜索关键词 encrypt crypt aes rsa des key iv 等等跟加密相关的。

方法三：结合各种参数、提示字符定位加密函数。如抓到的数据包中出行过的参数 ：source、key

方法四：根据 url 来定位。 网站登陆常常是向某一个 api 接口发送账号密码信息，因此点击提交时，用 js 发起请求就需要提供 api 地址。

然后ctrl+shift+F搜索api中的关键词。

0x03 常见几种加密方法

RSA

需要用到public_key，一般藏在隐藏的表单输入项中。

AES

需要用到key和iv。
